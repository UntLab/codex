Pre-Flight Checklist
1. Confirm login works for the assigned employee Use the staff email already created in Supabase Auth. If the employee can open index.html after login, the account is live.
2. Confirm the role badge shows the expected domains Shared users should see badges like Operations / Sales. Pure operations users should see only Operations.
3. Activate sender profiles only when ready Emails to clients and agents will use the assigned sales manager profile. Until SMTP credentials are saved in Settings, external delivery stays in fallback / safe mode.
4. Use the same browser session per tester Do not mix multiple employees in the same session during UAT. Log out between users to avoid confusing ownership and cached tokens.

Core UAT Scenarios
Scenario A: Shared Sales / Operations login Login with a shared user. Open quotation.html and operations.html. Expected result: both pages load, the role badge shows two domains, and there is no access error banner.
Scenario B: Operations-only protection Login with an operations-only user. Open quotation.html. Expected result: the page stays blocked with a clear message, while operations.html and settings.html remain available.
Scenario C: Request to quotation flow From quotation.html, claim a request, edit it, create a quotation, then move it to Booked. Expected result: a shipment file is created automatically and appears in Operations.
Scenario D: Operations handoff Open operations.html, take an unassigned shipment, edit core fields, save, then release it. Expected result: operation_manager_id changes correctly and the record stays visible in the proper queue.
Scenario E: Sender profile self-service Open settings.html under the logged-in employee. Expected result: the employee can view and edit only their own sender profile unless the user is Admin / HR.
Known Limitations
Real outbound delivery is not fully activated yet Quotation and agent-email flows are implemented, but real mailbox delivery still depends on configured SMTP credentials in Settings or a global fallback mailbox.
Google parity is still incomplete Core flow is live enough for internal testing, but CRM parity, document generation, finance parity, and some Google-era automations are still pending.
Use UAT with controlled records During the first internal rounds, use test clients / requests rather than business-critical live records until the team confirms the expected behaviour.
